## Usage Notes

### How this document should be created:

1. Human completes `spec.md` (including clarification loop)
2. Human asks LLM: *"Generate an implementation plan for this spec."*
3. LLM produces this document with section 13 (clarifications) filled
4. Human reviews and answers clarifications
5. LLM revises the plan based on answers
6. Human approves → LLM begins coding

### Rules:

- Do NOT write code until this plan is reviewed and approved
- Remove sections marked *(Optional)* if they don't apply
- The step-by-step plan (section 4) is the contract — deviations need justification
- Update this document if scope changes during implementation
